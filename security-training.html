When you type a username in the box, it will show you one of the following colours:

GREEN - Allowed to enter our base if they have the correct badge and motto.
Use the command :base to let them in.

YELLOW - CHECK FOR [NSC] Recruit This means the person doesn’t work at NSC.
You can send them to training if their motto says [NSC] Recruit by using the command :training

YELLOW - USE HR GATE This means that the person is HR+ member and should enter by using the HR gate in the lobby.
You can send them back to the lobby by saying the command :deny

RED - This means this member has either been removed due to inactivity or has been fired.
Do NOT let them into base, direct them to the TU desk.
GRAY -This means this member has been placed on BOS/NP/DNH.
Let a HR+ member know so they can contact Room Rights.

Now I’ll teach you how to use the security simulator. When a member appears on the rollers, say the command :gate and kindly greet them.
Then use the security system by typing their username in the box.
The easiest way to do this is to whisper the member and copy paste their username into the security search.
If our security system is ever down, ask the member to whisper you 3 NSC rules instead.
If a member is not wearing the correct badge or has the wrong motto, ask them to change it before letting them into base.
If they are wearing a Special Visitors badge, you can send them to the lobby and tell them to enter through the gate.

A short recap about the following commands:
:base (sends them to base)
:training (sends them to training)
:deny (sends back to lobby)
